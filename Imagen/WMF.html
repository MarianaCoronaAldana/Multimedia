WMF

¿Cuándo surge?

El Formato de Metarchivo de Windows, (WMF) es un formato de archivo gráfico diseñado por Microsoft para su sistema operativo (Windows) a principios de la década de 1990. Este formato no era independiente del dispositivo y podía manejar tanto gráficos vectoriales como mapas de bits.

Caracteristicas

  1. Es un formato de 16 bits introducido en Windows 3.0 del que existe una versión más reciente de 32 bits

  2. Es un formato de gráficos vectoriales que permite también la inclusión de mapas de bits.

  3. Se usa especialmente para la definición de ilustraciones y dibujos vectoriales.

  4. Mediante vectores, especifican las líneas, las áreas, colores y demás elementos gráficos

  5. Se usa principalmente para las ilustraciones y los dibujos, nunca para fotografías

  6. Admite colores RGB de 24 bits y puede utilizarse en la mayoría de las aplicaciones Windows.

Historia

  1. 1988: Aparecen los archivos WMF con el lanzamiento de Windows 2.0

  2. 1992: Microsoft publica la especificación de formato de archivo WMF con el lanzamiento de Windows 3.1

  3. 2005: Se informa de una vulnerabilidad de Microsoft por Symantec. En algunos casos, el motor de renderizado de gráficos permite la ejecución remote de código.

  4. Enero 2006: Se resuelve la vulnerabilidad en la actualización de seguridad en Microsoft TechNet (MS06-001).

  5. Septiembre 2006: Windows publica una nueva especificación del formato WMF.

  6. 1992: Microsoft introduce el formato de metarchivo mejorado (EMF) con Windows NT 3.1

  7. Actualmente: Microsoft lanzó EMF+ como una extensión del formato de archivo EMF existente.

Algoritmo de compresión de WMF

Es un formato de gráficos vectoriales que permite también la inclusión de mapas de bits. Esencialmente, un archivo WMF almacena una secuencia de llamadas a funciones que son enviadas al nivel gráfico de Windows GDI para regenerar la imagen. Es similar en propósito y diseño al formato PostScript usado en los sistemas operativos Unix.

Explicación

Un mapa de bits, imagen ráster o imagen de píxeles es una estructura o fichero de datos que representa una rejilla rectangular de píxeles o puntos de color. Las tareas más comunes de GDI son:
  1. El dibujo de líneas, curvas, polígonos.

  2. El relleno de cuadros, círculos, polígonos, etcétera.

  3. Igualmente se encarga del renderizado de fuentes y textos.

  4. El manejo de paletas.

Fórmula

Utiliza unidades metafile, donde una unidad es un twip (1/20 del punto tipográfico). Dado que el punto tipográfico = 1/72 pulgada, una unidad metafile o twip se corresponde con 1/1440 pulgada. Generalmente las aplicaciones generan ficheros WMF utilizando unidades propias que son mayores que el twip para evitar overflow
